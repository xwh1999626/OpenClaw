# Subtitle Workflow - 字幕生成工作流

> 牛叔专属：以后给他出 SRT 字幕的标准流程

## 触发条件
牛叔给一个音频文件（mp3/wav/m4a）→ 我出 SRT 字幕

## 流程

```
[音频输入]
    ↓
1. 语音识别（whisper-cli + ggml-small.bin）
    ↓
2. 转写结果 + 时间戳
    ↓
3. 【质量控制】专业名词核对
    - 拉术语表（glossary.md）
    - 不在表里的术语 → 去对应品牌官网核实
    - 牛叔要求的：每个品牌产品名必须去官网确认
    ↓
4. 按牛叔风格格式化（参考 ARRI Omnibar.srt 模板）
    - 短句断行
    - 中文标点
    - 保留英文品牌/技术词
    - 时间戳 HH:MM:SS,mmm
    - UTF-8 BOM
    ↓
5. 输出 .srt 文件
    ↓
6. 对比参考 SRT（如果有配对的 SRT 文件），验证输出质量
```

## 风格模板（基于 ARRI Omnibar.srt 提取）

### 单行长度
- 8-15 字为主
- 长句在自然停顿处断开（即使语法不完整）
- 例：`距离 ARRI 发布 Orbiter / 已经过去了7年`

### 标点
- 中文：`，。！？、——`
- 不加空格
- 例：`我们`（不是`我 们`）

### 数字处理
- 数字 + 中文单位：紧贴（`7年`、`20%`）
- 数字 + 英文单位：紧贴（`1,500 Lux`）
- 千分位：英文逗号（`1,500`）

### 英文术语
- 品牌名：保留英文（ARRI、INFINIBAR、TITAN）
- 技术词：保留英文（RGBAW、CRI、SSI、TM-30、⊿uv）
- 颜色：可中英混用（Amber琥珀色、Mint薄荷色）

### 时间戳
- 格式：`HH:MM:SS,mmm --> HH:MM:SS,mmm`
- 毫秒用**英文逗号**（不是点）

### 文件编码
- UTF-8 with BOM

## 质量检查清单

每次输出 SRT 前自查：

- [ ] 时间戳格式正确（逗号毫秒）
- [ ] 序号连续 1, 2, 3...
- [ ] 行长度 8-15 字（少数例外 OK）
- [ ] 中文标点不带空格
- [ ] 英文品牌/技术词保留
- [ ] 数字与单位紧贴
- [ ] 专业名词已对照 glossary.md 核实
- [ ] 文件带 UTF-8 BOM
